# HANDOFF — Bookings SaaS

> Phase 7 artifact. Hand-runnable by a second engineer (or a non-engineer for the "vibe-handoff" section). Generated by `bequite handoff` (post-implementation; v0.4.1).

## TL;DR for engineers

**What it is:** small-scale bookings SaaS (tenant → resource → slot → booking) with admin + customer roles.

**How to run it locally:**

```bash
# 1. Clone + install (assumes BeQuite repo as parent)
cd examples/01-bookings-saas/
bun install

# 2. Start Supabase locally
supabase start

# 3. Apply migrations + seed
supabase db reset --seed

# 4. Wire env vars (copy .env.example → .env.local)
cp apps/web/.env.example apps/web/.env.local
cp apps/api/.env.example apps/api/.env.local
# Edit each file; supabase start prints the local URL + anon key.

# 5. Boot
bun dev   # runs apps/web + apps/api in parallel via turbo

# 6. Visit http://localhost:3000

# 7. Sign in as admin (seeded user)
# email: admin@example.com
# magic-link will land in supabase Inbucket (http://localhost:54324)
```

**How to deploy:**

```bash
# 1. Provision Supabase (production project)
# 2. Provision Vercel project; link to apps/web
# 3. Provision Better-Auth (or Clerk if alt); set SECRET in Vercel + Supabase
# 4. supabase db push  # apply migrations to production
# 5. vercel --prod      # deploy web
# 6. Configure env vars in Vercel dashboard (Better-Auth secret, Supabase
#    keys, Resend API key, Inngest signing key)
# 7. Smoke-test: visit /, browse, attempt a booking
```

**Where the moving parts live:**

- `apps/web/` — Next.js (App Router) frontend
- `apps/api/` — Hono backend (deploys as Vercel Edge Function)
- `apps/web/src/styles/tokens.css` — design tokens (Doctrine Rule 1)
- `apps/web/src/lib/auth.ts` — Better-Auth config
- `packages/schemas/` — Zod schemas (shared client+server)
- `supabase/migrations/` — DB schema + RLS policies
- `supabase/functions/` — Edge functions (cancel reminders, etc.)
- `tests/walkthroughs/{admin,user}-walk.md` — Playwright walk docs
- `tests/e2e/` — Playwright specs (organized per role × viewport × locale)
- `tests/a11y/` — axe-core walks
- `scripts/{smoke,self-walk}.sh` — pre-deploy smoke tests
- `.github/workflows/{ci,axe}.yml` — CI gates

## Adding a feature (as a second engineer)

1. `bequite discover "<feature description>"` — clarifies scope.
2. `bequite plan` — generates spec + plan + phases + tasks.
3. For each task: `bequite implement <task-id>`. Check off in tasks.md.
4. `bequite verify` — must pass before PR.
5. `bequite review` — second-pass critique.
6. PR. CI runs the same `bequite verify` gates.
7. Merge. `bequite handoff` updates this file with the changes.

If `bequite verify` fails:
- Lint / typecheck — fix the file.
- Unit tests — narrow to the failing one; debug with logs.
- Playwright walk — visit the page locally; reproduce; fix or update walk if UX changed.
- axe — open `evidence/P6/axe/<role>/<route>.json` and address violations.

## TL;DR for non-engineers ("vibe-handoff")

This app helps customers book appointments at your business and helps your team manage availability and customers. The customer flow is:

1. Customer goes to your domain → sees a button "Book an appointment".
2. They pick a time → enter email + name → click confirm.
3. They get an email confirming the booking.
4. Day before the slot, they get a reminder.

You (the admin) can:

- Add resources (e.g., specific staff or rooms).
- Set when each resource is available.
- See all bookings in a dashboard.
- Cancel a booking if needed (the customer is notified).

**To grow:** add tenants (different businesses), or add features like custom intake forms (note: deferred — see "Non-goals" in spec).

**To pause/stop:** the app continues to work standalone; no monthly subscription is forced. If you want to hand it off to a different engineer, this file contains everything they need.

## Operating cost (estimate)

At 5,000 MAU per the scale tier:
- Vercel Pro: $20/mo
- Supabase Pro: $25/mo
- Better-Auth: $0 (self-hosted in apps/web)
- Resend (email): $0 free tier (3,000/mo); $20/mo if more
- Inngest: $0 free tier (50k steps/mo)
- Sentry: $0 free tier (5k events/mo); $26/mo if more
- PostHog: $0 free tier (1M events/mo)

**Total at 5,000 MAU: ~$45-90/mo** (depending on email + observability).

## Risk register

| Risk | Likelihood | Mitigation |
|---|---|---|
| Concurrent-booking race | Low | Row lock + 409 fallback (tested) |
| Resend outage → emails delayed | Medium | Inngest retries + dead-letter queue |
| RLS misconfiguration leaks data | Low | `bequite audit` Rule 10 enforces deny-by-default; tested. |
| Vercel function timeout (300s default) | Low | Pro tier extends to 800s; long jobs use Inngest |
| Supabase auth outage | Low | Better-Auth is self-hosted in apps/web; only DB depends on Supabase |
