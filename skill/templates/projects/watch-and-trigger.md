# template/projects/watch-and-trigger/

> Canonical scaffold for the most common automation use case: **watch X, when it changes, trigger Y.**
>
> Generated by `bequite init <project> --template=watch-and-trigger [--no-n8n]`.
> Article VIII (Constitution v1.1.0) binding. `polite_mode = true` by default.

## Polite-mode defaults (binding by `polite_mode = true`)

The generated `bequite.config.toml` includes:

```toml
[scraping]
polite_mode = true

# Auto-applied when polite_mode = true:
respect_robots_txt = true
rate_limit_per_domain = "1 req / 3 sec"
cache_ttl_hours = 24
cache_bust_requires_adr = true
stealth_libraries_blocked = true
captcha_solving_blocked = true
pii_aggregation_requires_consent_log = true
user_agent = "BeQuite-Scraper/{{VERSION}} (+{{MAINTAINER_EMAIL}})"
watch_budget_max_fires_per_week = 5
cost_alarm_warn_usd = 10
cost_alarm_block_usd = 100
```

To relax any of these → write an ADR documenting which rule + why.

## Per-target spec (`docs/automation-flows/<flow>.md` + `apps/scraper/src/targets/<target>.ts`)

Each target site gets a spec (Markdown) + an implementation file. Required sections:

```markdown
# Target: <domain.com>

## URL pattern
- Pattern: `https://<domain>.com/products/<sku>`
- Sample SKUs: <list>

## robots.txt summary
- robots.txt URL: https://<domain>.com/robots.txt
- Disallow rules relevant to our scrape: <list>
- Verdict: allowed / partially-allowed / forbidden

## Terms of Service summary
- ToS URL: <link>
- Relevant scraping clauses: <quote>
- Verdict: allows / forbids / silent (silent = ADR required)

## Polite-mode defaults
- Rate limit: 1 req / 3 sec  (default; bumping requires ADR)
- Cache TTL: 24h
- User-Agent: BeQuite-Scraper/<version> (+<maintainer-email>)

## Change-detection strategy
- Strategy: hash | JSON-Patch | structural | specific-field
- Field(s) to watch: <list>
- Diff sensitivity: <pixel-perfect | tolerant>

## Trigger
- Destination: n8n workflow `<flow-name>` | Slack webhook | DB table `<name>` | email <list>
- Idempotency key: <how computed>

## Watch-budget
- Expected fires per week: <number>
- max_fires_per_week (3× pause threshold): <number × 3>
- Action on exceed: pause-and-ask | suppress | log-only

## Test fixture
- Sample HTML: `tests/fixtures/<target>/sample-2026-05-10.html`
- Expected diff vs prior fixture: `tests/fixtures/<target>/expected-diff.json`

## ADRs touched
```

## n8n in `docker-compose.yml` by default (opt-out via `--no-n8n`)

Default scaffold ships n8n + Postgres + Redis as a 3-service compose stack:

```yaml
# infra/docker-compose.yml (excerpt)
services:
  n8n:
    image: n8nio/n8n:latest
    environment:
      - DB_TYPE=postgresdb
      - DB_POSTGRESDB_HOST=postgres
      - QUEUE_BULL_REDIS_HOST=redis
      - EXECUTIONS_MODE=queue
      - N8N_ENCRYPTION_KEY=${N8N_ENCRYPTION_KEY}
    ports: ["5678:5678"]
    depends_on: [postgres, redis]

  postgres:
    image: postgres:16
    environment:
      - POSTGRES_DB=n8n
      - POSTGRES_USER=n8n
      - POSTGRES_PASSWORD=${DB_PASSWORD}
    volumes: [pg_data:/var/lib/postgresql/data]

  redis:
    image: redis:7-alpine

volumes:
  pg_data:
```

Total RAM idle: ~400MB. Disk: ~500MB. Lights up on `docker compose up`.

The scraper writes its diff payload to a webhook (`http://n8n:5678/webhook/<flow>`), which n8n consumes per the committed `apps/n8n/workflows/<flow>.json`.

`--no-n8n` swaps `docker-compose.no-n8n.yml` (webhook-only; user wires their own consumer).

## bequite scrape doctor

Run before any production scrape kickoff:

```bash
bequite scrape doctor
```

Two checks combined into one report:

### Selector-drift detection

For each target in `apps/scraper/src/targets/`:

- Last 5 runs' hit counts (rows extracted).
- 3+ consecutive runs at 0 results → **flag as drifted**.
- Sudden change (rows < 50% of previous run) → **flag as drifted**.
- Drifted targets get a new task auto-filed in `state/task_index.json`.

### Cost projection

For each scrape configured to run:

- # URLs × Firecrawl-credits-per-page (when Firecrawl) or # URLs × bandwidth (when self-hosted) × proxy-cost-per-request (when proxies enabled).
- Warn if projected > `$10`.
- Block if projected > `$100` without ADR documenting the budget.

Output: `evidence/<phase>/scrape-doctor-<YYYY-MM-DD>.md` with per-target verdicts.

## Verification (Phase 6 gates added by Article VIII)

`bequite verify` adds these gates for watch-and-trigger projects:

1. **robots.txt fetched in live run** (not just config-flagged) — assert `robots.ts::fetchRobots()` was invoked and `Disallow` was honored.
2. **Rate-limit measured live** — over a 60-second window, ≤ 20 requests per domain (1 req / 3 sec).
3. **Cache hit ratio ≥ 50%** on second run within 24h (proves cache is wired).
4. **No PII fields in scraped output** without consent-log entry.
5. **No stealth lib in dependencies** unless ADR with `legitimate-basis` exists.
6. **No captcha-solver in dependencies** unless ADR with `legitimate-basis` exists.
7. **Watch-budget not exceeded** in last 7-day rolling window.
8. **`bequite scrape doctor` green** (no drifted targets; cost projection within budget).

## Forking guidance

To fork this template for a vertical-specific scaffold:

1. Copy `template/projects/watch-and-trigger/` → `template/projects/watch-and-trigger-<vertical>/`.
2. Pre-configure the `bequite.config.toml::scraping` block for your vertical's defaults.
3. Add vertical-specific compliance Doctrines (e.g. `fintech-pci` for payment-data scraping, `mena-pdpl` for MENA-region targets).
4. Pre-populate `docs/automation-flows/` with golden flows for the vertical.
5. Document forking rationale in the template's README.
