# BeQuite v1.0.0 — Layer 1 Harness Final

**Released: 2026-05-11**
**Maintainer: Ahmed Shawky ([@xpShawky](https://github.com/xpShawky))**

> **Build it right the first time. No chatter. No debug spirals.**

This is the v1.0.0 release of BeQuite — the first stable cut of the Layer 1 Harness (CLI + Skill + Memory Bank + Hooks + Doctrines + Templates). Everything that's been shipped from `v0.1.0` (foundation + Constitution v1.0.0) through `v0.20.5` (terminal exec via Studio API) is consolidated into v1.0.0.

The companion release [`v2.0.0-alpha.1`](./v2.0.0-alpha.1.md) marks the first pre-release of the Layer 2 Studio Edition (marketing site + dashboard + API).

## What's in v1.0.0

### The kernel — Layer 1 Harness

- **A versioned Constitution** — Iron Laws + forkable Doctrines. Now at v1.3.0 with **10 Articles**:
  - I. Specification supremacy
  - II. Verification before completion (banned weasel words)
  - III. Memory discipline (Cline-pattern six-file Memory Bank + state/ + receipts/)
  - IV. Security & destruction discipline + three-tier command-safety
  - V. Scale honesty
  - VI. Honest reporting
  - VII. Hallucination defense (PhantomRaven-class blocking)
  - VIII. Scraping & automation discipline (v0.5.1; ADR-009)
  - IX. Cybersecurity & authorized-testing discipline (v0.5.2; ADR-010)
  - X. Operational completeness — the "restart-the-build" rule (v0.16.0; ADR-014)

- **17 ADRs** captured. The most consequential:
  - ADR-008 (Master-file merge → two-layer architecture)
  - ADR-009 (Article VIII Scraping)
  - ADR-010 (Article IX Cybersecurity)
  - ADR-011 (CLI authentication — Phase-3 device-code per RFC 8628)
  - ADR-012 (Multi-model planning — manual-paste MVP, ToS-clean)
  - ADR-013 (Studio v2 architecture — 3 surfaces sharing one brand)
  - ADR-014 (Iron Law X — operational completeness)
  - ADR-015 (Studio API auth + write surface)
  - ADR-016 (Terminal execution Rules of Engagement — first endpoint that *executes* commands)

- **13 Doctrines** — `default-web-saas`, `cli-tool`, `ml-pipeline`, `desktop-tauri`, `library-package`, `ai-automation`, `fintech-pci`, `healthcare-hipaa`, `gov-fedramp`, `vibe-defense`, `mena-pdpl`, `eu-gdpr`, `mena-bilingual`. All forkable.

- **14 deterministic-gate hooks** — secret-scan, block-destructive, verify-package (PhantomRaven defense), pretooluse-pentest-authorization, pretooluse-no-malware, pretooluse-cve-poc-context, pretooluse-scraping-respect, posttooluse-{format,lint,audit}, stop-{verify-before-done, cost-budget}, sessionstart-{load-memory, cost-budget}.

- **A standalone Python CLI** (`bequite` + `bq` console scripts; `uvx`/`pipx`/`pip install` distributable post-PyPI publish):
  - 7-phase commands: `init`, `research`, `stack`, `plan`, `phases`, `tasks`, `implement`, `verify`, `review`, `handoff`
  - Per-host install: `skill install` (9 hosts: Claude Code, Cursor, Codex, Cline, Kilo, Continue, Aider, Windsurf, Gemini)
  - Audit + freshness: `audit`, `freshness` — Constitution drift detector + knowledge probe (npm, PyPI, crates.io, GitHub, OSV)
  - Auto-mode: `auto` — one-click P0→P7 with 8 safety rails (cost ceiling, wall-clock, banned words, failure threshold, hook-block, one-way-door, parallel-task fan-out per AkitaOnRails 2026)
  - Multi-model planning: `multi-model` — manual-paste workflow with 5 collaboration modes (Parallel / Specialist / Debate / Judge / Red-Team)
  - Receipts: `receipts {list,show,validate-chain,roll-up}` + `verify-receipts` (ed25519-signed chain)
  - Routing: `route {show,list,providers}` — multi-model cost-aware routing (Anthropic + OpenAI + Google + DeepSeek + Ollama)
  - Pricing: `pricing {refresh,show,list}` — best-effort live fetch + 24h cache + vendored fallback
  - Authentication: `auth {login,logout,whoami,status,refresh}` — local-file + offline + CI modes; device-code flow lands v0.20.x+
  - Exporters: `export --format {spec-kit-zip,claude-code-skill}`

- **20 personas** (the "you must act like" list):
  - Core 12: product-owner, research-analyst, software-architect, frontend-designer, backend-engineer, database-architect, qa-engineer, security-reviewer, devops-engineer, token-economist, skeptic, automation-architect
  - Specialty 5: scraping-engineer (Article VIII), security-auditor + pentest-engineer + cve-watcher + disclosure-timer (Article IX)
  - Multi-model trio: multi-model-planning-orchestrator + model-judge + red-team-reviewer (ADR-012)

- **Reproducibility receipts** — every phase emits a chain-hashed JSON receipt with model + prompt-hash + memory-snapshot-hash + diff-hash + tools-invoked + tests + cost. ed25519-signed per-project keypair (gitignored private; committed public). Auditable via `bequite verify-receipts`.

- **Three example projects** — `01-bookings-saas` (Next.js + Hono + Supabase; `default-web-saas` Doctrine), `02-ai-tool-wrapper` (Python click; `cli-tool` Doctrine), `03-tauri-note-app` (Tauri v2 + SvelteKit + SQLite + AES-GCM + OS keychain; `desktop-tauri` Doctrine).

- **125-test integration suite** (Python 3.14, all green): receipts (10), signing (9), router (15), pricing (14), e2e/seven-phase-walk (8), e2e/doctrine-loading (8), multi-model (12), auth (8), auto-state (13), exporters (8), skill-install (9), e2e/auto-mode (11).

- **Documentation** (8 docs, drafted v0.14.0): README, QUICKSTART, HOW-IT-WORKS, DOCTRINE-AUTHORING, HOSTS, AUTONOMOUS-MODE, SECURITY, MAINTAINER.

- **CI/release pipeline** (v0.15.0): five-job `ci.yml` (lint + test matrix Python 3.11–3.14 + audit + license-audit + pr-summary). `release.yml` fires on `v*.*.*` tag (PyPI Trusted Publisher OIDC + GitHub Release with extracted notes). `commitlint.yml` enforces Conventional Commits.

### The companion — Studio Edition (Layer 2)

Layer 2 ships in parallel as the [`v2.0.0-alpha.1`](./v2.0.0-alpha.1.md) pre-release. It includes:

- **Marketing site** (`studio/marketing/`) — Next.js 15 cinematic landing + 6 deep MDX vibecoder tutorials + brand system (gold-on-black palette + Geist + Framer Motion + R3F-ready scaffold).
- **Dashboard** (`studio/dashboard/`) — Next.js 15 operations console with dual-mode loader (filesystem ↔ HTTP), LiveIndicator (SSE-driven `router.refresh()`), and live xterm.js terminal in HTTP mode.
- **API** (`studio/api/`) — Hono on Bun back-end. Three-mode auth (local-dev / token / device-code), append-only write surface (receipts + snapshots), four SSE event streams, terminal exec surface (allow-list-gated per ADR-016).

3D astronaut models via Blender pipeline are explicitly **descoped** from v1.0.0 + v2.0.0-alpha.1 per Ahmed's direction. The R3F scaffold remains in place (`studio/marketing/components/three/AgentCharacter3D.tsx` + `public/3d/` directory) so a future contributor can drop GLB models in without restructuring.

## Honest scope per Article VI

What v1.0.0 does **not** include:

- **3D Blender pipeline** — explicitly descoped. R3F scaffold preserved for future contribution.
- **Bidirectional WebSocket terminal** — POST-exec + SSE-output works for the 90% case (`bequite auto`); live stdin forwarding lands v0.21.0+ if interactive use cases emerge.
- **Better-Auth full integration** — Bearer-token MVP works for single-machine + multi-agent dev. Better-Auth lands v0.20.x+ once the multi-user auth-server topology is in place.
- **Postgres mirror** — the API is filesystem-backed today. Postgres lands v2.0.0 when multi-user / cloud operation is live.
- **PyPI + npm publishes** — the `release.yml` workflow is wired; tag-triggered. Ahmed pushes the v1.0.0 PyPI wheel manually (one-way door).

What v1.0.0 verified in this session:

- API + dashboard both `tsc --noEmit` exit 0.
- 125/125 Python integration tests green on Python 3.14.
- Constitution v1.3.0 + 17 ADRs + 13 Doctrines + 14 hooks + 20 personas all consistent across CLAUDE.md / AGENTS.md / activeContext.md / progress.md.
- Live boot smoke for the Studio surfaces deferred (Bun runtime not in this developer's environment).

## Migration / breaking-change notes

v1.0.0 is the first stable release. There are no breaking changes from v0.20.5 — the 1.0 tag marks "API surface stabilized for the v1.x line."

For projects that were tracking pre-1.0:

- **Constitution version**: bump from 1.0.0 → 1.3.0 if you forked v0.1.0 or earlier; the v1.3.0 changes are additive (Articles VIII / IX / X added; no Article removed or relaxed).
- **Receipt schema**: stable since v0.7.0. ed25519 signing optional but recommended.
- **CLI command surface**: stable since v0.10.7. Nothing renamed; only additions in v0.11.0 → v0.20.5.
- **Doctrine schema**: stable since v0.1.1. mena-bilingual added in v0.11.0 (was deferred from v0.1.1).
- **Hook surface**: stable since v0.5.2. Hooks added by Article IX (v0.5.2) are opt-in via Doctrine activation.
