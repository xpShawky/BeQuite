# BeQuite Studio Edition — v2.0.0-alpha.1

**Released: 2026-05-11**
**Status: First pre-release — alpha-quality**
**Maintainer: Ahmed Shawky ([@xpShawky](https://github.com/xpShawky))**

> **The visual surface for BeQuite. Three Next.js + Hono apps sharing one gold-on-black brain.**

This is the first pre-release of **Layer 2 Studio Edition** — the operating system that runs on top of the Layer 1 Harness ([`v1.0.0`](./v1.0.0.md)). Studio reads what the Harness writes (Memory Bank + state + receipts + cost ledger) and exposes it as a cinematic landing page, an operations dashboard, and a Hono API back-end.

This is `alpha.1` because:

- The full multi-user / cloud topology (Postgres mirror, Better-Auth-backed sessions, Kubernetes deploy) lands in `v2.0.0` stable.
- The HTTP-mode swap is end-to-end working but not yet smoke-tested in a Bun-equipped CI environment.
- The 3D astronaut Blender pipeline is descoped per Ahmed's direction.

What `v2.0.0-alpha.1` ships is the foundation: brand system + marketing + dashboard + API + auth + writes + SSE streams + terminal exec. Every piece is on `main` and tested via `tsc --noEmit` exit 0. Live boot smoke from a clean clone is the next verification step.

## Highlights

### Cinematic marketing site (`studio/marketing/`)

- **Apple-grade pinned-scroll** primitive (`components/Chapter.tsx`) — `useScroll` + `MotionValue<number>` render-prop pattern, the same shape as Apple's macbook-pro page.
- **Hero with character drift** — gold astronaut zen-pose, scale + opacity transforms driven by scroll progress.
- **7-phase reveal** — each phase activates as the user scrolls past it; opacity / x / scale per phase index.
- **6 deep MDX tutorials** at `/docs/*` — `quickstart` (10 min zero-to-first-receipt), `from-scratch` (full salon-bookings worked example), `retrofit` (installing into existing project), `multi-model-planning` (5 collaboration modes), `auto-mode` (8 safety rails + recovery), `troubleshooting` (vibecoder pain points + fixes). ~250-290 lines each.
- **R3F starfield** — 800 gold motes drifting through 3D space; respects `prefers-reduced-motion`.
- **Brand-consistent dark theme** with gold accents (`--gold-primary: #E5B547`).

### Operations dashboard (`studio/dashboard/`)

- **Image-6-mock layout** — TopBar + PhasesSidebar (P0..P7 with status icons + DEV STATUS + DEPLOY) + center main + AgentPanel (right-side astronaut + status + activeContext).
- **Dual-mode loader** — `BEQUITE_DASHBOARD_MODE=filesystem|http`. Server components `await loadProject()` and get the same `ProjectSnapshot` shape either way.
- **LiveIndicator** — `"use client"` pill (LIVE / CONNECTING / STALE / OFFLINE / FS) with throttled `router.refresh()` on SSE events. The dashboard updates in real-time without manual reload (Iron Law X step 7 = "no" by construction).
- **xterm.js Terminal** — brand-themed (gold cursor on ink-pure), Run / Cancel / Clear toolbar. Sends `X-BeQuite-RoE-Ack: ADR-016` on every exec. Stdin disabled in alpha.1 (per ADR-016 §6).
- **Receipts list** — last 10 receipts with ed25519 signature badges + cost + timestamp.
- **Brand-consistent dark theme** — same tokens as marketing.

### Studio API (`studio/api/`)

- **Hono on Bun** — ~3MB bundle when built; cold start <100ms.
- **Three auth modes** via `BEQUITE_AUTH_MODE`:
  - `local-dev` (default) — pass-through. `X-BeQuite-Auth-Mode` header surfaces the mode.
  - `token` — Bearer + sha256-on-disk at `<workspace>/.bequite/.auth/tokens.json`. Mint via `POST /api/v1/auth/token`.
  - `device-code` — RFC 8628 stub returning 503 until v0.20.x+ auth server.
- **Read surface** — `/api/v1/projects`, `/api/v1/projects/snapshot`, `/api/v1/receipts/{,:sha}`, `/api/v1/snapshots/:version`. Path-traversal guard via `BEQUITE_WORKSPACE_ROOT` allow-list.
- **Append-only write surface** (per ADR-015) — `POST /api/v1/receipts` (idempotent on content-hash) + `POST /api/v1/snapshots` (refuses overwrite). Iron Law X attestation block on every write.
- **Four SSE event streams** (per v0.20.0) — `/api/v1/streams/{all,receipts,cost,phase}`. Reference-counted file watcher per workspace; 30s heartbeat; clean teardown on client abort.
- **Terminal exec surface** (per ADR-016) — `POST /api/v1/terminal/exec` with `X-BeQuite-RoE-Ack: ADR-016` required. Hardcoded allow-list (`bequite`, `bq` only). 10MB output ring buffer. SIGTERM-then-SIGKILL on cancel/timeout. Per-execution receipt for full audit trail.

### Iron Law X — Operational completeness

Every write endpoint returns an `iron_law_x` block (ADR-014):

```json
{
  "persisted_path": "...",
  "file_readable": true,
  "file_size_bytes": 1234,
  "file_sha256": "...",
  "api_route_alive": true,
  "attestation": "Operationally complete. ...",
  "caller_must": ["..."]
}
```

If the re-read fails or the sibling probe returns non-ok, the helper **throws** and the route returns 500. There is no "should work" path. Banned weasel words (`should`, `probably`, `seems to`, `appears to`, `I think it works`, `might work`) **never** appear in the attestation string.

## Quickstart — three terminals

```bash
# Terminal 1 — start the API on :3002
cd studio/api
bun install
bun run src/index.ts
# → BeQuite Studio API v2.0.0-alpha.1 listening on http://localhost:3002

# Terminal 2 — start the dashboard on :3001 in HTTP mode
cd studio/dashboard
pnpm install   # or npm install
BEQUITE_DASHBOARD_MODE=http pnpm dev
# → http://localhost:3001 — full live operations console

# Terminal 3 — start the marketing site on :3000
cd studio/marketing
pnpm install
pnpm dev
# → http://localhost:3000 — cinematic landing + /docs tutorials
```

If you only want the dashboard (no API), unset `BEQUITE_DASHBOARD_MODE` and the loader uses filesystem-mode (single-machine dev). The LiveIndicator surfaces this as `FS` instead of `LIVE`.

## Honest scope per Article VI

What `v2.0.0-alpha.1` does **not** include:

- **3D Blender pipeline.** The R3F scaffold (`components/three/AgentCharacter3D.tsx` + `public/3d/` directory) remains in place for future contribution but is not wired. A future contributor can drop GLB models in without restructuring.
- **Better-Auth integration.** Bearer-token MVP works for single-machine + multi-agent dev. Better-Auth (per Doctrine `default-web-saas` Rule 9) lands once the multi-user auth-server topology is finalized.
- **Postgres mirror.** The API is filesystem-backed today. Postgres lands in `v2.0.0` stable when multi-user / cloud operation goes live.
- **Live stdin forwarding for the terminal.** Per ADR-016 §6, deferred to `v0.21.0+` if interactive use cases emerge.
- **Bidirectional WebSocket transport.** SSE-output + POST-exec covers 95% of use cases. WS bidirectional is a v0.21.0+ exploration.
- **Full Bun-equipped CI smoke.** Both `studio/api/` and `studio/dashboard/` typecheck clean (`tsc --noEmit` exit 0); live-boot end-to-end smoke is the next verification step before `v2.0.0` stable.
- **Public-network deployment story.** v2.0.0-alpha.1 is single-machine dev or trusted-LAN. TLS termination + IP-allow-list + production-grade auth lands with the auth server.

## Migration / breaking-change notes

This is the first pre-release of the Layer 2 Studio Edition. There is no prior 2.x to migrate from.

**For Layer 1 Harness users** (`v1.0.0` users): nothing changes. The Harness CLI keeps working filesystem-only. Studio is opt-in. If you don't need the dashboard, you don't need to install Bun, Next.js, or any TypeScript dependency.
