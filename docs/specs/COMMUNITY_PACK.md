# Community Pack — `/bq-community` (C15) — alpha.24

Plan a community for a course, brand, creator, or launch. Command: `bq-community.md`. Skills: product-strategist + writing-dna + localization-rtl.

## Why a command (not an argument)

Shared by ≥3 parents (course / launch / brand) — a single home beats duplicating the same argument across three commands. Distinct 11-artifact planning set.

## Outputs — `.bequite/community/`

COMMUNITY_STRATEGY · MEMBER_PERSONA · PLATFORM_DECISION · ONBOARDING_FLOW · CONTENT_AND_EVENT_CALENDAR · ENGAGEMENT_LOOPS · MODERATION_RULES · ROLES_AND_PERMISSIONS · GROWTH_PLAN · RISKS · NEXT_STEPS.

## Discipline

Platform chosen by where members already are + moderation needs + MENA habits (WhatsApp-first common); engagement loops + weekly rhythm over vanity metrics; moderation + spam prevention + roles from day one; health measured by active participation, not member count; don't build a custom platform when Discord/Telegram/WhatsApp fits. Practical structure, not motivational fluff. **Built alpha.24 — NOT live-tested.**
